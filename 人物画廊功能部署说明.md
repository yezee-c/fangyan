# 人物画廊功能部署说明

## 一、功能概述

本次开发为若依框架新增了完整的**人物画廊**功能,包括:

### 1. 后端功能
- 人物信息管理(CRUD)
- 人物文件关联(图片、视频、文档)
- 公开访问接口(无需登录)
- MinIO文件存储集成

### 2. 若依后台管理
- 人物列表页面 (`/system/character/info`)
- 人物编辑页面 (`/system/character/edit`)
- 文件关联管理

### 3. 独立前端展示(gallery-app)
- 人物画廊列表 (`/characters`)
- 人物详情页 (`/character/:id`)
- 文件展示页 (`/gallery`)

## 二、数据库部署

### 1. 创建人物画廊表
```bash
mysql -u root -p ry-vue < sql/character_gallery.sql
```

### 2. 添加菜单权限
```bash
mysql -u root -p ry-vue < sql/character_menu.sql
```

执行后会在系统中创建以下菜单:
- 人物画廊(ID: 3100) - 顶级菜单
- 人物管理(ID: 3101) - 子菜单
- 相关权限按钮(ID: 3102-3106)

## 三、后端部署

### 1. 确认MinIO配置

检查 `ruoyi-admin/src/main/resources/application.yml`:

```yaml
# MinIO配置
minio:
  endpoint: http://localhost:9000
  accessKey: minioadmin
  secretKey: minioadmin
  bucketName: ruoyi
```

### 2. 启动MinIO服务

```bash
# Windows
minio.exe server D:\minio-data

# Linux
./minio server /data/minio
```

访问 http://localhost:9000 确认MinIO正常运行

### 3. 启动若依后端

```bash
cd ruoyi-admin
mvn clean package
java -jar target/ruoyi-admin.jar
```

## 四、若依前端部署

### 1. 安装依赖(如果还未安装)
```bash
cd ruoyi-ui
npm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

访问 http://localhost:80 登录后台

### 3. 使用人物管理功能

1. 登录若依后台
2. 左侧菜单找到 **人物画廊 > 人物管理**
3. 点击"新增"按钮创建人物
4. 填写基本信息后保存
5. 编辑人物,添加图片、视频等文件

## 五、独立前端部署(gallery-app)

### 1. 安装依赖

```bash
cd gallery-app
npm install
```

如果npm install遇到权限错误,可以:
- 以管理员权限运行PowerShell
- 或使用 `npm install --force`

### 2. 启动开发服务器

```bash
npm run serve
```

访问 http://localhost:8081

### 3. 页面说明

- `/gallery` - 公共文件展示(图片、视频预览)
- `/characters` - 人物画廊列表
- `/character/:id` - 人物详情页

## 六、功能测试

### 1. 测试人物管理

#### 后台管理端:
1. 登录若依后台
2. 进入 **人物画廊 > 人物管理**
3. 新增人物,填写姓名、头衔、简介等
4. 保存后编辑人物

1. 访问 http://localhost:8081/characters
2. 查看人物卡片列表
3. 点击卡片进入详情页
4. 查看人物信息、图片集、视频集

### 2. 测试公开访问

- 人物画廊列表接口: `GET /system/character/gallery/list`
- 人物详情接口: `GET /system/character/gallery/detail/{id}`
- 这两个接口使用了 `@Anonymous` 注解,无需登录即可访问

### 3. 测试文件关联

1. 先在 **文件管理 > 公共文件** 中上传图片或视频
2. 在人物编辑页点击"添加图片"或"添加视频"
3. 从文件列表中选择文件
4. 可以设置文件分类和说明
5. 确认后文件会关联到人物

## 七、重要接口说明

### 管理端接口(需要权限):
- `GET /system/character/list` - 查询人物列表
- `GET /system/character/{id}` - 获取人物详情
- `POST /system/character` - 新增人物
- `PUT /system/character` - 修改人物
- `DELETE /system/character/{ids}` - 删除人物
- `POST /system/character/files` - 添加文件关联
- `DELETE /system/character/files/{id}` - 删除文件关联

### 公开访问接口(无需权限):
- `GET /system/character/gallery/list` - 获取公开人物列表
- `GET /system/character/gallery/detail/{id}` - 获取人物详情

## 八、注意事项

### 1. 文件上传流程
1. 必须先在"文件管理"中上传文件
2. 然后在人物编辑页面关联已上传的文件
3. 不支持在人物编辑页面直接上传新文件

### 2. 头像和封面
- 头像和封面需要在文件管理中上传后,获取文件ID
- 在人物编辑页填入对应的文件ID
- 后续版本可优化为直接选择文件

### 3. 文件类型限制
- 图片: image/*
- 视频: video/*
- 文档: application/*, text/*

### 4. MinIO访问
- 默认使用预签名URL,有效期7天
- URL包含认证参数,可直接访问
- 图片和视频可在浏览器中预览

## 九、路由配置

### 若依后台路由(需手动添加)

在若依后台的路由文件中需要添加以下路由:

```javascript
{
  path: '/system/character',
  component: Layout,
  hidden: true,
  children: [
    {
      path: 'info',
      component: () => import('@/views/system/character/info'),
      name: 'CharacterInfo',
      meta: { title: '人物管理', activeMenu: '/system/character' }
    },
    {
      path: 'edit',
      component: () => import('@/views/system/character/edit'),
      name: 'CharacterEdit',
      meta: { title: '编辑人物', activeMenu: '/system/character' }
    }
  ]
}
```

### 独立前端路由

已自动配置在 `gallery-app/src/router/index.js`

## 十一、故障排查

### 1. 后端启动失败
- 检查数据库连接
- 检查MinIO配置
- 查看控制台错误日志

### 2. 前端无法访问后端
- 检查代理配置 `vue.config.js`
- 确认后端服务已启动
- 查看浏览器Network面板

### 3. 文件上传失败
- 检查MinIO服务是否运行
- 检查bucket是否存在
- 查看后端日志错误信息

### 4. 图片不显示
- 检查MinIO URL是否正确
- 查看预签名URL是否过期
- 确认文件确实存在于MinIO中
